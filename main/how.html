factorB

Как это работает?

Сервис основан на сравнении хешей денежных требований. Применение хешей и соответствующих функций хеширования позволяет минимизировать риск утраты коммерческой / банковской тайны. Подробнее о функции хеширования см.ниже.
Для проведения теста достаточно выполнить три действия:

Действие 1

Факторинговая компания самостоятельно хеширует каждое денежное требование по инструкции сервиса. Итогом хеширования является создание уникального хеша для каждого денежного требования. Полученные хеши сохраняются в сводном файле.

Действие 2

Факторинговая компания загружает сводный файл в сервис сверки factorB.

Действие 3

factorB сверяет все накопленные хэши разных факторинговых компаний, выявляет совпадения и высылает отчет на указанный при загрузке e-mail адрес.

Порядок действий подробно описан в инструкции, доступной после нажатия клавиши "Начать использовать сервис".

Хэширование

Решение предлагаемое factorB основано на применении алгоритмов хеширования – преобразование по определенному алгоритму входного массива данных произвольной длины в выходную битовую строку фиксированной длины - хеш.



Для хеширования нет необходимости использовать стороннее ПО.
Все операции выполняются с помощью стандартных функций любой системы учёта факторинговых операций и общедоступных алгоритмов хеширования.

Хеширование выполняется силами ФК.
Первичные данные, представляющие коммерческую / банковскую тайну, ни на каком этапе не передаются ни по открытым, ни по шифрованным каналам связи и остаются исключительно в ФК.

Ключевое свойство хэширования - шифрование возможно только в одном направлении. Обратно расшифровать хэш невозможно. Это свойство хеширования является одним из ключевых элементов ЭЦП.
Дополнительное свойство хеширования - одному набору данных соответствует строго одно значение хеша.

Таким образом, зная хеш каждого денежного требования, сервис factorB может найти одинаковые денежные требования в портфелях факторинговых компаний, не о имея доступа к коммерческой / банковской тайне.

06cbbd047944cec151f2c4115b71636dc6510e4d14a363ab51ad862abfced59e

Пример хеша для набора символов "Клиент ООО «Сахар», Дебитор ООО «АШАН», Дата 01/01/2016 №001 Сумма 100.00 Вата сахарная сто тонн"

c69d979a8a963914342ab79874a515cbc26d6a3ad368038c8af10a74cfcf3441

Важно отметить, что зная хеш, нереально не только расшифровать изначальное значение (даже зная алгоритм шифрования), но и практически нереально предположить какой хеш к каким данным относится.
Подробнее применяемых алгоритмах, защите от методов взлома, включая "радужные" таблицы см. в инструкции.
