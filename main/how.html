Порядок действий

Действие 1

Факторинговая компания (ФК) самостоятельно хеширует каждое денежное требование. Итогом хеширования является создание уникального хеша для каждого денежного требования. Все хеши сохраняются в итоговом файле.


Действие 2

Факторинговая компания загружает итоговый файл на сторону factorB.


Действие 3

factorB сверяет все накопленные хэши и высылает отчет на указанный e-mail адрес.

Хэширование

Решение предлагаемое factorB основано на применении алгоритмов Хеширования – преобразование по определенному алгоритму входного массива данных произвольной длины в выходную битовую строку фиксированной длины - хеш.



Для хеширования нет необходимости использовать стороннее ПО, все операции выполняются с помощью стандартных функций любой системы учёта факторинговых операций и общедоступных алгоритмов хеширования. Подробнее см. инструкцию.

Все хеширование выполняется силами ФК. Первичные данные ни на каком этапе не передаются ни по открытым, ни по шифрованным каналам связи и остаются исключительно в ФК.

Свойством хэширования является то, что шифрование возможно только в одном направлении. Обратно расшифровать хэш практически нереально.  Поэтому хеширование является одним из ключевых элементов ЭЦП. В России - ГОСТ Р 34.11-2012. Одному набору данных соответствует строго одно значение хеша.
